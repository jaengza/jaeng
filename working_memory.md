## 📌 1. ภาพรวมของโปรเจกต์ (Project Overview)
**kesineTrader** เป็นเว็บแอปพลิเคชันการเงินและการเทรดระดับพรีเมียม (High-Tech Premium Trading Dashboard) สำหรับนักเทรดกลุ่ม ICT (Inner Circle Trader) โดยมีฟีเจอร์หลัก 2 ส่วนคือ:

2. **AI News Feed & Market Analytics**: ฟีดข่าวสารด่วนจากตลาดคริปโตและทองคำ พร้อมระบบแปลเนื้อหาและสรุปข่าวเป็นภาษาไทยโดยอัตโนมัติในพื้นหลัง (Background AI Translation & Summarization)

## ⚙️ 3. ระบบ Auto-Planner (ICT / RSI / Fibonacci OTE)
ระบบวางแผนเทรดบูรณาการการวิเคราะห์จากทิศทางเทรนด์ (Swing High/Low), ระดับความตึงตัวของราคา (RSI), และสเกลราคา Fibonacci:

### 📐 โครงสร้างผลลัพธ์ (Split Layout)
* ผลลัพธ์การวิเคราะห์จะเรนเดอร์ในโหมด **Side-by-Side (เคียงข้างกัน)** บน Desktop ภายใน `#autoplan-analysis-center` ประกอบด้วย:
  * **ฝั่งซ้าย**: **Smart Trading Card (ICT Card)** แสดงป้ายทิศทาง (BUY/SELL), อัตรา Risk-to-Reward (R:R), โซนราคา TP, Entry, SL, ขนาด Lot Size และเกจ RSI
  * **ฝั่งขวา**: **Fibonacci Scale Line** แสดงสเกลราคาและระดับเปอร์เซ็นต์สำคัญทางจิตวิทยา (0.0%, 38.2%, 50.0%, OTE Zone 62.0% - 79.0%, 100.0%)

## 🌐 4. ระบบ AI News Feed & แปลภาษาไทย (Translation System)
ระบบดึงข่าวสารภาษาอังกฤษด่วนผ่าน API และดำเนินการสรุปความพร้อมแปลเป็นภาษาไทยในระดับ Client-side ทันที:

### 🧪 สถาปัตยกรรมระบบแปล (Background Translation)
1. **การโหลดแบบ Non-blocking**: ข่าวภาษาอังกฤษจะดาวน์โหลดและแสดงผลทันทีพร้อม Badge แสดงสถานะและวงกลมไฟกะพริบ (`กำลังสรุปเป็นภาษาไทย...` 🟡) เพื่อให้หน้าเว็บไม่ค้าง
2. **การดึงข้อมูลแปล**: ระบบใช้ Client-side API Bridge (Google Translate Client Bridge) ในไฟล์ `js/market.js` เพื่อแปล Title และ Body (ซึ่งมักจะเป็น Summary ย่อส่วนจาก API) ไปพร้อมกัน
3. **การแสดงผลราบรื่น**: เมื่อได้รับการแปลเสร็จสิ้น หน้าจอจะเปลี่ยนจากภาษาอังกฤษเป็นสรุปภาษาไทยทันทีด้วยแอนิเมชัน **Fade-in & Blur filter transition** ที่สวยงาม
4. **ระบบแคชอัจฉริยะ (Local Storage Cache)**:
   * หลังแปลเสร็จสิ้น หัวข้อและคำแปลจะถูกนำไปแคชใน `localStorage` ของผู้ใช้ โดยผูกกับ ID ข่าว
   * หากเปิดเว็บซ้ำ ข่าวชิ้นเดิมจะโหลดสรุปภาษาไทยและตรา `แปลและสรุปโดย AI 🇹🇭` สดใสในทันทีโดยไม่ต้องรอแปลซ้ำอีกครั้ง

### 🗂️ การผนวกเข้ากับแผงข้อมูลสนับสนุนขวา (Collapsible Right Drawer Integration)
1. **ราคาเรียลไทม์ในสองตำแหน่ง**: ระบบเชื่อมต่อตัวแปร DOM ทั้งของหน้าจอเทรดหลักและ `#mini-btc-price` / `#mini-xau-price` ใน Drawer ทำให้ทุก ๆ วินาทีที่มี WebSocket tick จาก Binance หรือการ Polling ทองคำ ราคาบนหน้าจอหลักและขวาสุดจะอัปเดตและกะพริบไฟสีเขียว/แดงพร้อมกัน
2. **ข่าวด่วนฉบับย่อ 5 อันดับแรก (Drawer Mini News Feed)**: 
   * ดึงข่าว 5 ข่าวล่าสุดจากรายการข่าวทั้งหมด โดยจัดระเบียบตามลำดับเวลา และให้ความสำคัญ (Priority) กับข่าวสารที่แปลสรุปเป็นภาษาไทยเรียบร้อยแล้วขึ้นก่อนเพื่อให้ผู้ใช้งานอ่านง่ายที่สุด
   * เชื่อมโยง ID สำหรับแปลอัตโนมัติ ทำให้เมื่อข่าวบนหน้าจอหลักแปลเสร็จสมบูรณ์ ข่าวฉบับย่อใน Drawer จะเปลี่ยนเป็นภาษาไทยพร้อมตราสัญญลักษณ์ `AI 🇹🇭` และแอนิเมชันทันที
   * สามารถกด ยุบเก็บ หรือ สลับเรียกดูแผงขวานี้ได้อย่างอิสระบนเดสก์ท็อปผ่านปุ่ม Sidebar Toggle หรือปุ่มสี่เหลี่ยมด้านมุมซ้ายล่างสุด

## ⚡ 6. ระบบ Caching & Fast-Fail Timeout (v3.2)
เพื่อเพิ่มประสิทธิภาพและความลื่นไหลขั้นสูงสุดให้กับการวิเคราะห์อัตโนมัติ ระบบได้ติดตั้งตรรกะใน [market.js](file:///g:/My%20Drive/jaeng/js/market.js) ดังนี้:
1. **Smart Local Caching:** ผลลัพธ์ข้อมูลแท่งเทียน OHLCV ที่ดึงมาจาก API ทุกตัว (ทองคำ ทองคำ DXY และ BTC) จะถูกเก็บไว้ในหน่วยความจำชั่วคราว (`ohlcvCache`) เป็นระยะเวลา 15 วินาที หากมีการเรียกวิเคราะห์สินทรัพย์ตัวเดิมหรือกดปุ่มซ้ำ ระบบจะส่งข้อมูลจากหน่วยความจำทันที (~1ms) โดยไม่ต้องต่อเน็ตใหม่ ลดปริมาณ Network requests และแก้ไขการทำงานช้าได้ 100%
2. **Fast-Fail Timeout Wrapper:** ครอบลิงก์เชื่อมต่อทั้งหมดด้วย JavaScript `AbortController` โดยกำหนด Timeout สำหรับ Proxy เครือข่ายที่ 2.5 วินาที และ API อื่นที่ 2.0 วินาที เพื่อจำกัดอาการดีเลย์จากอินเทอร์เน็ต และดึง Fallback/Simulated Data มาแสดงผลทดแทนโดยหน้าเว็บไม่ค้าง

## ⚡ 9. ข้อมูลทางวิศวกรรมการอัปเกรดระบบ (v3.7, v3.8, และ v3.9 Update Details)

### 📌 A. ระบบคำนวณต้นทุนสเปรดมาตรฐาน Exness (v3.7)
เพื่อวิเคราะห์ต้นทุนการเทรดที่แท้จริง ระบบได้นำตรรกะคำนวณและแสดงผลต้นทุนสเปรด (Estimated Spread Cost) ของ Exness ในรูปแบบสีม่วงพาสเทลพรีเมียม เข้าสู่ระบบคำนวณความเสี่ยงและวิเคราะห์Confluence:
1. **สูตรคำนวณมาตรฐาน (Spread Cost standard Formula):**
   $$\text{ต้นทุนสเปรด (USD)} = \text{ระยะสเปรด (Points)} \times \text{ขนาด 1 Point} \times \text{ขนาดสัญญา (Contract Size)} \times \text{ขนาด Lot}$$
2. **ค่าคงที่อ้างอิงฐานข้อมูล Exness (Exness Constants Setup):**
   * **XAUUSD (ทองคำ):** Point = 0.01 | ขนาดสัญญา = 100 | ค่าสเปรดเฉลี่ย = 200 Points | **ตัวคูณ Lot = 1.0**
   * **BTCUSDT (บิตคอยน์):** Point = 0.01 | ขนาดสัญญา = 1 | ค่าสเปรดเฉลี่ย = 2,150 Points | **ตัวคูณ Lot = 0.01 (ล็อตต่ำสุดคือ 0.01 เสมอ)**
   * **ETHUSDT (อีเธอเรียม - ปัจจุบันปิดการโหลดราคา):** Point = 0.01 | ขนาดสัญญา = 1 | ค่าสเปรดเฉลี่ย = 160 Points | ตัวคูณ Lot = 0.01
   * **FOREX (คู่เงินทั่วไป):** Point = 0.0001 | ขนาดสัญญา = 100,000 | ค่าสเปรดเฉลี่ย = 15 Points | **ตัวคูณ Lot = 10.0**
3. **การแสดงผล Net Profit:** นำเสนอมิติการวิเคราะห์ "กำไรสุทธิ (หักสเปรด)" ควบคู่ไปกับแผนเทรดเพื่อให้นักเทรดตระหนักถึงต้นทุนการดำเนินงานของ Exness เสมอ

### 📅 B. การขยายช่วงเวลา 1 วัน & 1 สัปดาห์ และปุ่มกดเลือก Ergonomic (v3.8)
เพิ่มมิติการเทรดระยะยาวด้วยการรองรับช่วงเวลา 1 วัน (1d) และ 1 สัปดาห์ (1w) พร้อมปรับปรุงขนาดปุ่มให้หรูหรากดง่ายแบบ Mobile-First:
1. **การซิงก์กราฟ TradingView (String Interval Sync):** ปลดล็อกคำสั่ง `parseInt()` ใน `bindChart()` ของ [js/app.js](file:///g:/My%20Drive/jaeng/js/app.js) ช่วยให้แถบ Advanced Widget สามารถรับส่ง Interval เป็นตัวอักษรดิบได้อย่างถูกต้อง เช่น สลับกราฟ Daily ('D') และ Weekly ('W') โดยหน้าจอไม่ค้างหรือไม่เป็นหน้าว่าง
2. **ข้อมูลแท่งเทียนระยะยาว (Yahoo & Binance Mapping):**
   * ปรับปรุงให้ Yahoo Finance API ทำการแมปช่วงกว้างประวัติย้อนหลังของสัปดาห์ (`1w`) เป็น `'1wk'` และขยายการดึงข้อมูลประวัติย้อนหลังเป็น **5 ปี (`'5y'`)** ค้ำประกันแท่งเทียนสมบูรณ์สำหรับการดึงกลับมาวิเคราะห์อย่างน้อย 250 แท่ง
   * ปรับปรุงให้ Binance API สามารถดึงข้อมูลสัปดาห์ `'1w'` ได้ราบรื่น และจำลองช่วงเวลาจำลองใน `market.js` ให้ยอมรับสัปดาห์ที่ `604,800,000` มิลลิวินาที
3. **ดีไซน์ปุ่มสัมผัสพรีเมียม (Premium Pill Style):**
   * ปรับแต่งสไตล์ของปุ่มช่วงเวลา `.ctrl-tf`, `.tf-pill`, และ `.chart-tf-btn` / `.chart-sym-btn` ให้ขยายพื้นที่สัมผัสเป็น `padding: 7px 15px`, กำหนดความสูงขั้นต่ำ `min-height: 36px` และมุมโค้ง `border-radius: 6px`
   * ขยายฟอนต์ของปุ่มเป็น `0.78rem` ตัวหนาพิเศษ `700` ทำให้สัมผัสด้วยนิ้วมือบนสมาร์ทโฟนได้ง่ายยิ่งขึ้นและสวยงามสมส่วนพรีเมียม
   * จัดแถบกราฟเป็น 6 ปุ่มสมดุล (`5m`, `15m`, `1h`, `4h`, `1d`, `1w`) โดยถอด `1m` ออกเพื่อให้สมมาตรกับหน้าจอพกพา

### 🔬 C. การแบคเทสย้อนหลัง 30 วัน ฐานทุนคงที่ $50 และการกู้ตรรกะล็อตระดับวิกฤต (v3.9)
ติดตั้งระบบทดสอบคุณภาพระดับสากลและแก้ไขบั๊กลับของระบบคำนวณพอร์ตที่มีความเสี่ยงต่ำ:
1. **การค้นพบบั๊กคำนวณล็อต (Critical Lot Size Bug Fixed):**
   * *ปัญหาเดิม:* ระบบทำการปัดเศษทศนิยมล็อตลงก่อนด้วย `Math.floor(lot * scale) / scale` แล้วจึงเช็คขีดจำกัดขั้นต่ำสุด (`if lot > 0 && lot < minLot`) ส่งผลให้ในพอร์ตขนาดเล็ก ($50) และระยะ SL กว้าง ค่าล็อตดิบที่น้อยกว่าขั้นต่ำ (เช่น 0.00166) จะโดนปัดลงเป็น `0` ก่อน ทำให้เข้าเงื่อนไขไม่ผ่าน และหน้าจอจะแแสดงล็อตเป็น `--`
   * *การแก้ไขใน v4.0:* ย้ายการตรวจสอบและล็อคขีดจำกัดขั้นต่ำขึ้นมาดำเนินการก่อนการปัดเศษทศนิยม โดยปรับขีดจำกัด Lot ขั้นต่ำสุดของทุกสินทรัพย์ (รวมถึง BTC) ให้เป็น **0.01** เสมอ ทำให้พอร์ตขนาดเล็กแสดงล็อตขั้นต่ำ 0.01 ได้อย่างเสถียร 100%
2. **การแบคเทสย้อนหลัง 30 วัน ฐานทุนคงที่ $50 และกำหนดขั้นต่ำ 0.01 Lot (Fixed Start Risk Backtest):**
