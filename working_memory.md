# 🧠 ApexTrader Hub - Working Memory & Project Documentation

เอกสารฉบับนี้เป็นไฟล์ **"ความจำในการทำงาน (Working Memory)"** ของโปรเจกต์ **ApexTrader Hub** เพื่อบันทึกภาพรวมสถาปัตยกรรม โครงสร้าง และข้อมูลทางเทคนิคสำคัญทั้งหมด ช่วยให้สามารถทำความเข้าใจ เรียนรู้ และพัฒนาแอปพลิเคชันต่อยอดได้อย่างรวดเร็วและถูกต้องสมบูรณ์แบบ

---

## 📌 1. ภาพรวมของโปรเจกต์ (Project Overview)
**ApexTrader Hub** เป็นเว็บแอปพลิเคชันการเงินและการเทรดระดับพรีเมียม (High-Tech Premium Trading Dashboard) สำหรับนักเทรดกลุ่ม ICT (Inner Circle Trader) โดยมีฟีเจอร์หลัก 2 ส่วนคือ:
1. **Auto-Planner (ICT/RSI/Fibo)**: ระบบวางแผนการเทรดอัจฉริยะแบบบูรณาการที่คำนวณหา Fibonacci OTE (Optimal Trade Entry) พร้อม Risk & Lot Size Management
2. **AI News Feed & Market Analytics**: ฟีดข่าวสารด่วนจากตลาดคริปโตและทองคำ พร้อมระบบแปลเนื้อหาและสรุปข่าวเป็นภาษาไทยโดยอัตโนมัติในพื้นหลัง (Background AI Translation & Summarization)

## ⚙️ 3. ระบบ Auto-Planner (ICT / RSI / Fibonacci OTE)
ระบบวางแผนเทรดบูรณาการการวิเคราะห์จากทิศทางเทรนด์ (Swing High/Low), ระดับความตึงตัวของราคา (RSI), และสเกลราคา Fibonacci:

### 📐 โครงสร้างผลลัพธ์ (Split Layout)
* ผลลัพธ์การวิเคราะห์จะเรนเดอร์ในโหมด **Side-by-Side (เคียงข้างกัน)** บน Desktop ภายใน `#autoplan-analysis-center` ประกอบด้วย:
  * **ฝั่งซ้าย**: **Smart Trading Card (ICT Card)** แสดงป้ายทิศทาง (BUY/SELL), อัตรา Risk-to-Reward (R:R), โซนราคา TP, Entry, SL, ขนาด Lot Size และเกจ RSI
  * **ฝั่งขวา**: **Fibonacci Scale Line** แสดงสเกลราคาและระดับเปอร์เซ็นต์สำคัญทางจิตวิทยา (0.0%, 38.2%, 50.0%, OTE Zone 62.0% - 79.0%, 100.0%)

## 🌐 4. ระบบ AI News Feed & แปลภาษาไทย (Translation System)
ระบบดึงข่าวสารภาษาอังกฤษด่วนผ่าน API และดำเนินการสรุปความพร้อมแปลเป็นภาษาไทยในระดับ Client-side ทันที:

### 🧪 สถาปัตยกรรมระบบแปล (Background Translation)
1. **การโหลดแบบ Non-blocking**: ข่าวภาษาอังกฤษจะดาวน์โหลดและแสดงผลทันทีพร้อม Badge แสดงสถานะและวงกลมไฟกะพริบ (`กำลังสรุปเป็นภาษาไทย...` 🟡) เพื่อให้หน้าเว็บไม่ค้าง
2. **การดึงข้อมูลแปล**: ระบบใช้ Client-side API Bridge (Google Translate Client Bridge) ในไฟล์ `js/market.js` เพื่อแปล Title และ Body (ซึ่งมักจะเป็น Summary ย่อส่วนจาก API) ไปพร้อมกัน
3. **การแสดงผลราบรื่น**: เมื่อได้รับการแปลเสร็จสิ้น หน้าจอจะเปลี่ยนจากภาษาอังกฤษเป็นสรุปภาษาไทยทันทีด้วยแอนิเมชัน **Fade-in & Blur filter transition** ที่สวยงาม
4. **ระบบแคชอัจฉริยะ (Local Storage Cache)**:
   * หลังแปลเสร็จสิ้น หัวข้อและคำแปลจะถูกนำไปแคชใน `localStorage` ของผู้ใช้ โดยผูกกับ ID ข่าว
   * หากเปิดเว็บซ้ำ ข่าวชิ้นเดิมจะโหลดสรุปภาษาไทยและตรา `แปลและสรุปโดย AI 🇹🇭` สดใสในทันทีโดยไม่ต้องรอแปลซ้ำอีกครั้ง

### 🗂️ การผนวกเข้ากับแผงข้อมูลสนับสนุนขวา (Collapsible Right Drawer Integration)
1. **ราคาเรียลไทม์ในสองตำแหน่ง**: ระบบเชื่อมต่อตัวแปร DOM ทั้งของหน้าจอเทรดหลักและ `#mini-btc-price` / `#mini-xau-price` ใน Drawer ทำให้ทุก ๆ วินาทีที่มี WebSocket tick จาก Binance หรือการ Polling ทองคำ ราคาบนหน้าจอหลักและขวาสุดจะอัปเดตและกะพริบไฟสีเขียว/แดงพร้อมกัน
2. **ข่าวด่วนฉบับย่อ 5 อันดับแรก (Drawer Mini News Feed)**: 
   * ดึงข่าว 5 ข่าวล่าสุดจากรายการข่าวทั้งหมด โดยจัดระเบียบตามลำดับเวลา และให้ความสำคัญ (Priority) กับข่าวสารที่แปลสรุปเป็นภาษาไทยเรียบร้อยแล้วขึ้นก่อนเพื่อให้ผู้ใช้งานอ่านง่ายที่สุด
   * เชื่อมโยง ID สำหรับแปลอัตโนมัติ ทำให้เมื่อข่าวบนหน้าจอหลักแปลเสร็จสมบูรณ์ ข่าวฉบับย่อใน Drawer จะเปลี่ยนเป็นภาษาไทยพร้อมตราสัญญลักษณ์ `AI 🇹🇭` และแอนิเมชันทันที
   * สามารถกด ยุบเก็บ หรือ สลับเรียกดูแผงขวานี้ได้อย่างอิสระบนเดสก์ท็อปผ่านปุ่ม Sidebar Toggle หรือปุ่มสี่เหลี่ยมด้านมุมซ้ายล่างสุด

## 🎨 5. ระบบดีไซน์ระดับพรีเมียมและความสะอาดตา (Premium & Large UI Design)
หน้าเว็บถูกดีไซน์ใหม่ทั้งหมดตามหลัก **"Ultra-Readability UI"** เน้นความสะอาดตา ข้อมูลครบถ้วน และขนาดที่ใหญ่ชัดเจน:

### 🌟 ตัวแปรสีพรีเมียมหลัก (Dark Cyberpunk Theme)
* `--bg-base`: `#06080c` (พื้นหลังดำลึกพรีเมียม)
* `--bg-panel`: `rgba(13, 17, 26, 0.65)` (กระจกขุ่น Blur strength 16px)
* `--accent-cyan`: `#00f2fe` (แรเงาสีฟ้าสะท้อนวิสัยทัศน์)
* `--accent-gold`: `#f5a623` (สีทองอร่ามแจ้งเตือนความสำคัญ)
* `--accent-emerald`: `#10b981` (สีเขียว TP)
* `--accent-red`: `#f43f5e` (สีแดง SL / ขาดทุน)

### 📈 กฎการปรับสเกลขนาดใหญ่พิเศษ (Scale-up Configurations)
* **กล่องข้อมูล (Inputs & Selects)**: ขยายเป็น `padding: 14px 18px` และ `font-size: 1.05rem` เพื่อให้กดและเลือกได้ง่ายขึ้นอย่างโดดเด่น
* **ปุ่มสั่งการ (Buttons)**: ขยายเป็น `padding: 16px 20px`, `font-size: 1.05rem`, และปรับมุมโค้งมนพรีเมียมเป็น `border-radius: 12px`
* **ข้อมูลตัวเลขราคา OTE (Prices in Smart Card)**: ขยายราคา TP, Entry, SL เป็น `font-size: 1.35rem` ด้วยน้ำหนักฟอนต์หนาพิเศษ `800` พร้อมแถบสีเขียว ฟ้า แดง คมชัด อ่านง่ายมาก
* **ล็อตไซส์ (Lot Size Metric)**: ขยายตัวเลขสัญญาขนาดใหญ่เด่นเป็น `1.25rem`
* **ข่าวสารสรุปแปลไทย (News Feed)**: 
  * หัวเรื่องข่าวภาษาไทยขยายเป็น `1.15rem` หรือ `1.2rem` (หนา `700` ตัวหนาพิเศษ) พร้อมเพิ่ม `line-height: 1.45` ป้องกันตัวหนังสือชนกัน
  * คำสรุปเนื้อหาแปลไทยขยายเป็น `0.95rem` และ `line-height: 1.6` เว้นระยะห่างบรรทัดสวยงาม สะอาดตา อ่านลื่นไหลอย่างแท้จริง

## ⚡ 6. ระบบ Caching & Fast-Fail Timeout (v3.2)
เพื่อเพิ่มประสิทธิภาพและความลื่นไหลขั้นสูงสุดให้กับการวิเคราะห์อัตโนมัติ ระบบได้ติดตั้งตรรกะใน [market.js](file:///g:/My%20Drive/jaeng/js/market.js) ดังนี้:
1. **Smart Local Caching:** ผลลัพธ์ข้อมูลแท่งเทียน OHLCV ที่ดึงมาจาก API ทุกตัว (ทองคำ ทองคำ DXY และ BTC) จะถูกเก็บไว้ในหน่วยความจำชั่วคราว (`ohlcvCache`) เป็นระยะเวลา 15 วินาที หากมีการเรียกวิเคราะห์สินทรัพย์ตัวเดิมหรือกดปุ่มซ้ำ ระบบจะส่งข้อมูลจากหน่วยความจำทันที (~1ms) โดยไม่ต้องต่อเน็ตใหม่ ลดปริมาณ Network requests และแก้ไขการทำงานช้าได้ 100%
2. **Fast-Fail Timeout Wrapper:** ครอบลิงก์เชื่อมต่อทั้งหมดด้วย JavaScript `AbortController` โดยกำหนด Timeout สำหรับ Proxy เครือข่ายที่ 2.5 วินาที และ API อื่นที่ 2.0 วินาที เพื่อจำกัดอาการดีเลย์จากอินเทอร์เน็ต และดึง Fallback/Simulated Data มาแสดงผลทดแทนโดยหน้าเว็บไม่ค้าง

## 📱 7. Responsive Mobile Layout Architecture (v3.3)
สถาปัตยกรรมการแสดงผลบนอุปกรณ์เคลื่อนที่ใน [style.css](file:///g:/My%20Drive/jaeng/css/style.css) ได้รับการปรับปรุงอย่างลึกซึ้งเพื่อให้ราคาทศนิยม 3 ตำแหน่งของดัชนีดอลลาร์ DXY และ XAU ไม่หลุดขอบหรือตกหล่น:
1. **Header Asset Columns:** ในจอมือถือ (max-width: 767px) ปรับ `.asset-btn` ให้เรียงตัวแนวตั้ง (`flex-direction: column`) บีบช่องว่างและ padding ให้แคบลง ส่งผลให้มีระยะขอบในแนวนอนที่กว้างขวาง ป้องกันไม่ให้ปุ่ม DXY ซึ่งอยู่ขวาสุดเลื่อนตกหรือล้นออกขอบขวา
2. **Dashboard Price Chips:** ภายใต้ `@media (max-width: 580px)` ลด padding ชิปราคาลงเหลือ `6px 8px` และกำหนดขนาดตัวเลขราคา `.chip-price` ให้ปรับขนาดยืดหยุ่นตามความกว้างจอ ป้องกันอักขระและเปอร์เซ็นต์ดันกรอบการ์ด Glassmorphism
3. **Fibonacci OTE price spacing:** กำหนดความกว้างคงที่ของ `.fibo-price` ที่ `85px` และระบุ `min-width: 0` ให้กับ `.fibo-track` แถบสเกล เพื่อป้องกันราคาทศนิยม 3 ตำแหน่งเบียดและทับซ้อนกับแถบพลัง
4. **Grid-based 2-Line ICT Items:** บนจอมือถือแคบ (ต่ำกว่า 580px) ปรับโครงสร้าง `.ict-item` เป็น Grid 2 แถว นำ Label ขึ้นไปแสดงบรรทัดบนสุด และเว้นพื้นที่แถวด้านล่างให้กับราคา (.ict-item-value) และ Badge สถานะได้อย่างสมมาตร แก้ไขข้อมูลล้นขอบขวาของจอได้ 100%
